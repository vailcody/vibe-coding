Контекст — главное оружие

AI хорош настолько, насколько хорош контекст, который вы ему даёте

@Files / @Folders

Подключайте конкретные файлы и папки к запросу. AI видит их содержимое и учитывает при генерации.

@Codebase

Поиск по всему проекту. AI найдёт релевантные файлы сам. Полезно для: «найди где используется X».

@Docs

Подключите документацию библиотек. AI будет писать код по актуальной документации, а не по устаревшим знаниям.

@Web

Поиск в интернете прямо из чата. Свежие решения, обновления библиотек, Stack Overflow.

MCP

Model Context Protocol — расширения для AI-агента

Что это?

Стандартный протокол, который даёт AI-агенту доступ к внешним инструментам: базы данных, API, файловые системы, браузер.

Как подключить

.cursor/mcp.json — один JSON-файл. Cursor автоматически подхватит все описанные серверы и покажет доступные инструменты.

Полезные MCP

  • 🗄 PostgreSQL — агент пишет и проверяет запросы
  • 🌐 Browser — агент видит UI приложения
  • 📋 Figma — агент читает макеты
  • 📁 Filesystem — расширенная работа с файлами

Результат

Агент не просто пишет код — он проверяет результат. Написал SQL → выполнил → увидел ошибку → исправил. Полный цикл.

15

Экономия токенов

Как не сжечь бюджет за день

  • Edit mode для мелких правок — дешевле агента в 5-10 раз
  • Конкретный контекст — @file лучше @codebase (меньше токенов, точнее результат)
  • Декомпозиция — маленькие задачи = маленький контекст = дешевле
  • Новый чат — длинная история съедает токены. Закончил задачу → новый чат
  • Модели для разных задач — Claude для архитектуры, быстрые модели для рутины
  • Rules — не повторяйте одно и то же в каждом промпте
16

Когда агент ломает

Стратегии выхода из тупика

🔄

Зацикливание

Агент исправляет ошибку и создаёт новую. Решение: остановить, описать проблему заново, дать контекст ошибки явно.

⏪

Checkpoint / Restore

Cursor сохраняет состояние после каждого шага. Откатитесь к рабочему состоянию и попробуйте другой подход.

🧹

Чистый контекст

Если чат слишком длинный — начните новый. Скопируйте только суть проблемы. Свежий взгляд = свежий результат.

🧠

Человек в цепи

Не давайте агенту бесконтрольно менять код. Ревью каждого diff, approve по частям. Вы — архитектор, AI — исполнитель.

17

Best Practices

  • Промпт = ТЗ. Чем чётче описали задачу — тем лучше результат
  • Итеративно. Не пытайтесь сделать всё за один промпт. Маленькие шаги
  • Git. Коммитьте перед каждым крупным запросом к агенту
  • Ревью. Всегда читайте diff. AI генерирует уязвимости и ненужный код
  • Не доверяйте слепо. AI уверенно врёт. Проверяйте библиотеки, API-эндпоинты
