Цитата Карпаты — мощный якорь. Разделить для двух аудиторий: разрабы и аналитики.

⚡ Возможности

  • 🎨 Быстрое прототипирование — от идеи до рабочего UI за минуты
  • 🔄 Автоматизация рутины — парсинг, отчёты, скрипты
  • 🛠️ Внутренние инструменты — без очереди к разработчикам
  • 🧪 Генерация тестов, документации, код-ревью
  • 📊 Личная эффективность — анализ данных, автоматизация документов
Примеры близкие к аудитории — банковский софт, внутренние инструменты, тесты.

⚠️ Ограничения

  • Не заменяет разработчика на сложных архитектурных задачах
  • Garbage in → garbage out: нужно понимать, что хочешь получить
  • Контекстное окно ограничено — большие проекты нужно декомпозировать
  • Безопасность: AI может генерить уязвимый код — ревью обязательно
  • Галлюцинации: иногда «выдумывает» API и библиотеки
Важно для разработчиков — показать что мы трезво оцениваем технологию, не хайпим бездумно.

📐 Методология: сверху вниз

1
Подготовка спецификации
ТЗ, промты, требования — фиксируем «что хотим» до кода
2
Прототип в браузерном агенте
Имитация функционала — прокликали, увидели corner cases
3
Скачиваем код → оживляем локально
Доработка в IDE с AI-агентом — у него уже есть контекст
Разработчики оценят — это по сути Spec-Driven Development. Прототип даёт агенту контекст — структуру данных, визуал, логику.

🌐 Почему начинаем с браузерных?

  • Низкий порог входа — не нужно настраивать окружение
  • Быстрый результат — вау-эффект за минуты
  • Встроенные UI-инструкции — красивый результат из коробки
  • Кодовая база = контекст — для локального агента на следующем шаге
  • Дешёвая проверка — протыкали, нашли corner cases, поняли механику
Для разработчиков: подчеркнуть что кодовая база из браузерного агента — это контекст (данные, UI, структура), который упрощает работу локальному агенту.
